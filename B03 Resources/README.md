# B03 Resources

> **Reference material, studies, permanent authored notes.**

## What goes here

- **studies/** — technical and non-technical learning
- **investment/** — personal finance (optional)
- Authored notes that do not fit a specific project
- Durable references (guides, cheatsheets)
- Summarized books (when they do not fit in C03)

## Suggested structure

```
B03 Resources/
├── studies/                  (technical and non-technical learning)
├── investment/               (personal finance, optional)
└── <your-domains>/
```

*Adapt to your domains. The folder is loose by design.*

## Difference from A02 Anchor Topics

- **A02 Anchor** = high-level MOC, thesis + link map
- **B03 Resource** = specific, concrete note that can be referenced by anchors

Example:
- `A02 Anchor Topics/Observability.md` (MOC with your thesis)
- `B03 Resources/studies/Development/Backend/logs-vs-metrics.md` (specific note)

## Rule

**Authorship.** Notes in B03 are written by YOU. The AI can:
- Search references that complement
- Suggest links to other notes
- Propose restructuring

The AI does NOT write content in B03 without review.

## When to create a B03 note

- It already left A01 with a mature idea
- It has permanent value (not a task, not an event)
- You will want to reference it later

## Maintenance checklist

- [ ] Quarterly review — are the notes still true?
- [ ] Links to/from anchors in A02
